# v9 噪点问题分析与 v10 优化方案

## 一、现象归纳

- **Front 路**：道路结构清晰，无大片模糊和竹叶噪声；存在**局部噪点**（散落颗粒/闪烁）。
- **Back 路**：**持续满屏噪点**，几乎看不清场景。
- **部分帧**：**四周大片连续噪点**。

## 二、根因分析

### 1. 为何 Back 路满屏噪点？

- **视角约束弱**：车体向前运动，后方视角在时间上主要是“被拉远”，同一 3D 区域来自后方视角的**多视角、大基线**观测少，几何与外观约束都弱于前向。
- **球谐 (SH) 过拟合**：view-dependent 颜色用 SH 拟合。在约束不足的方向（尤其是 back）上，SH 容易拟合到噪声或高频变化，渲染出**闪烁、颗粒状**的“满屏噪点”。
- **低透明度高斯残留**：约束不足的区域容易留下大量半透明、细小的高斯，渲染叠加后看起来像持续噪点。

数据上：v9 过滤后 BACK 仍有 50–53 帧/路，**帧数不少**，问题主要来自**几何/视角结构**，而不是单纯“帧数少”。

### 2. 为何 Front 路有局部噪点？

- **小浮游体**：原动态物体或边缘区域，在 mask 或几何不确定处，容易产生很多**小、半透明**高斯，形成局部颗粒/闪烁。
- **边缘约束弱**：画面四周往往被更少视角覆盖，容易产生**四周连续噪点**。
- ** densify 过猛**：梯度大的地方（边缘、遮挡、反光）会不断分裂出新高斯，若 cull 不狠，会留下大量“噪点式”小高斯。

### 3. 为何有的帧“四周大片连续噪点”？

- 某些时刻的**相机位姿**或**场景覆盖**使画面边缘在训练中得到的多视角约束更少。
- 边缘 + 弱约束 → 同一套参数下，这些帧的**边缘区域**更容易出现成片噪点。

## 三、优化思路（与参数对应）

| 方向 | 手段 | 作用 |
|------|------|------|
| 去低透明度噪点 | 提高 **cull-alpha-thresh**（0.01 → **0.02**） | 更积极剔除半透明高斯，直接减颗粒/满屏噪点 |
| 减 SH 闪烁 | 降低 **sh-degree**（3 → **2**） | 减弱 view-dependent 过拟合，缓解 back 路闪烁/颗粒 |
| 少生小高斯 | 提高 **densify-grad-thresh**（0.0008 → **0.0012**） | 降低在不确定区域分裂新点的强度，减局部噪点 |
| 限制细长高斯 | **max-gauss-ratio**（5 → **4**） | 尺度正则更严，减少“针状”高斯带来的噪点感 |

以上均在**保持 v9 优点**（去大片模糊、去竹叶噪声、道路结构清晰）的前提下，只做**适度收紧**，避免重新引入模糊或结构崩塌。

## 四、v10 具体方案

- **数据**：继续使用 v9 数据集（`scene-0061_s0_v9`），不再删帧。
- **训练**：在 v9 参数基础上仅做下表修改，其余（如 stop-screen-size-at、cull-scale-thresh、use-scale-regularization 等）与 v9 一致。
- **脚本**：新增 `run_train_v10.sh`，experiment 名为 `scene-0061_s0_v10`。

### v10 参数变更表

| 参数 | v9 | v10 | 说明 |
|------|----|-----|------|
| `cull-alpha-thresh` | 0.01 | **0.02** | 提高透明度剔除，去噪点 |
| `sh-degree` | 3 | **2** | 降低 SH 阶数，减 back 闪烁 |
| `densify-grad-thresh` | 0.0008 | **0.0012** | 减少在不确定区域分裂 |
| `max-gauss-ratio` | 5.0 | **4.0** | 更严尺度比，减针状/颗粒感 |
| 其余 | — | 同 v9 | 不改 |

### 预期效果

- **Back 路**：满屏噪点减轻，至少可辨道路与大致结构。
- **Front 路**：局部噪点减弱，四周连续噪点减少。
- **整体**：仍保持道路结构清晰、无大片模糊、无竹叶噪声；若 v10 偏糊或细节损失过多，再单独微调 cull-alpha-thresh 或 sh-degree。

## 五、实施步骤

1. 新增并运行 `run_train_v10.sh`（30k 步，数据目录 v9）。
2. 训练结束后用最新 checkpoint 渲染 `scene-0061_sunny_v10.mp4`。
3. 对比 v9/v10：重点看 back 路满屏噪点、front 局部与边缘噪点是否改善，以及是否引入模糊或细节损失。
